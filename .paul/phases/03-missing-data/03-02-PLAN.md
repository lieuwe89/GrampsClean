## AC-1: Living/deceased filter scopes scan results
```gherkin
Given the Missing Data tab has a filter dropdown showing "All people"
When the user selects "Living only" and clicks Scan
Then only people without a death event appear in the result list
 And people with a death event are excluded regardless of missing fields

When the user selects "Deceased only" and clicks Scan
Then only people with a death event appear in the result list
 And people without a death event are excluded
```

## AC-2: Status bar shows per-field breakdown
```gherkin
Given a scan completes with results
When the final status is displayed
Then the status bar shows the total count AND a per-field breakdown
 For example: "42 people — 12 missing Birth Date, 8 missing Death Date"
 And fields with zero missing people are omitted from the breakdown
```

## AC-3: Double-clicking a row navigates to the person in GRAMPS
```gherkin
Given the scan has produced results in the Missing Data tab
When the user double-clicks a result row
Then the GRAMPS main window navigates to that person
 And the People view becomes active (same pattern as IslandTab)
 And rows with no handle (e.g. "no results" placeholder) do nothing
```

    **Navigation — add `_navigate_person` method to MissingTab:**
    Copy the exact pattern from IslandTab (02-03 SUMMARY):

    ```python
    def _navigate_person(self, handle):
        """
        Navigate the GRAMPS main window to the given person.
        Follows the same pattern as IslandTab._navigate_person.
        """
        if not handle:
            return
        self._uistate.set_active(handle, 'Person')
        try:
            vm = self._uistate.viewmanager
            cat_num = vm.get_category("People")
            if cat_num is not None:
                vm.goto_page(cat_num, None)
        except Exception:
            pass
    ```

    **Wire navigation into ResultList:**
    In `__init__`, change the ResultList instantiation from:
    ```python
    self._result_list = ResultList(COLUMNS)
    ```
    to:
    ```python
    self._result_list = ResultList(COLUMNS, on_activate=self._navigate_person)
    ```

    **Detailed status counts — in `_scan_fn()`:**
    After collecting `rows`, compute per-field counts, then build the status
    string that replaces the simple `f"{total} people with missing data"`:

    ```python
    total = len(rows)
    # Count occurrences of each field label across all rows
    field_counts = {}
    for row in rows:
        for label in row[2].split(", "):
            if label:
                field_counts[label] = field_counts.get(label, 0) + 1

    # Build status: "42 people — 12 missing Birth Date, 8 missing Death Date"
    FIELD_ORDER = ["Birth Date", "Death Date", "Birth Place", "Death Place"]
    parts = [
        f"{field_counts[f]} missing {f}"
        for f in FIELD_ORDER
        if f in field_counts
    ]
    status = f"{total} people — " + ", ".join(parts) if parts else f"{total} people with missing data"
    ```

    Update the yield loop to use the new `status` variable.
    The yield structure itself remains identical:
    ```python
    for i, row in enumerate(rows):
        s = status if i == len(rows) - 1 else ""
        yield row, s
    ```

    Do NOT change the `if not rows` early-exit or any other part of _scan_fn.
