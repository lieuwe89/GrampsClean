  <action>
    Create grampsclean/tab_islands.py.

    Imports needed (plain names — sys.path already patched by tool.py loader):
    - gi / Gtk (gi.require_version first)
    - from widgets import ResultList
    - from worker import ScanWorker
    - from graph import KinshipGraph

    **COLUMNS** module constant:
    ```python
    COLUMNS = [("Name", 0), ("Group Size", 1), ("Detail", 2)]
    ```

    **IslandTab(Gtk.Box):**

    __init__(self, db_wrap):
    - Call Gtk.Box.__init__(orientation=VERTICAL, spacing=6)
    - set_border_width(8)
    - Store self.db_wrap = db_wrap
    - self._max_size = 10  # default

    Build toolbar (Gtk.Box, spacing=6, border_width=4):
    - Scan button: Gtk.Button(label="Scan"), connect "clicked" → _on_scan
    - Cancel button: Gtk.Button(label="Cancel"), connect "clicked" → _on_cancel
      - set_sensitive(False) initially (greyed out)
    - Gtk.Label(label="Max group size:")
    - SpinButton: Gtk.SpinButton.new_with_range(1, 50, 1)
      - set_value(10)
      - connect "value-changed" → _on_max_size_changed
    - pack_start toolbar items: scan_btn (False,False,0), cancel_btn (False,False,0),
      then pack separator space, then label + spin right-aligned using pack_end

    Store: self._scan_btn, self._cancel_btn, self._spin

    Build result list:
    - self._result_list = ResultList(COLUMNS)
    - pack_start(toolbar, False, False, 0)
    - pack_start(self._result_list, True, True, 0)

    Create worker (not started yet):
    - self._worker = ScanWorker(self._result_list, self._scan_fn, self.db_wrap)

    **_on_scan(self, btn):**
    - If self._worker.is_running(): return
    - Recreate worker (fresh scan): self._worker = ScanWorker(self._result_list, self._scan_fn, self.db_wrap)
    - self._scan_btn.set_sensitive(False)
    - self._cancel_btn.set_sensitive(True)
    - Connect worker completion detection via GLib.timeout_add(200, self._check_done)
    - self._worker.start()

    **_on_cancel(self, btn):**
    - self._worker.cancel()
    - self._cancel_btn.set_sensitive(False)

    **_on_max_size_changed(self, spin):**
    - self._max_size = int(spin.get_value())

    **_check_done(self) -> bool:**
    - If self._worker.is_running(): return True (keep polling)
    - self._scan_btn.set_sensitive(True)
    - self._cancel_btn.set_sensitive(False)
    - return False (stop timeout)

    **_scan_fn(self, db_wrap) -> generator:**
    - This is the scan logic, called by ScanWorker in a background thread
    - Build graph: graph = KinshipGraph(db_wrap); graph.build()
    - Get islands: islands = graph.get_islands(self._max_size)
    - Count totals: total_people = sum(len(c) for c in islands)
    - total_groups = len(islands)
    - row_count = 0
    - For each component in islands (sorted smallest first — already sorted):
      - group_size = len(component)
      - detail = "Isolated" if group_size == 1 else f"Group of {group_size}"
      - For each handle in sorted(component):  # sorted for deterministic order
        - person = db_wrap.get_person_from_handle(handle)
        - if person is None: continue
        - name = db_wrap.format_name(person)
        - row_count += 1
        - yield (name, str(group_size), detail), ""
    - Final status (yielded as last item's status, but worker uses last status_text):
      Actually the worker uses the LAST status_text from any yield.
      So yield a sentinel at the end:
      status = f"{total_people} people in {total_groups} groups (max size: {self._max_size})"
      yield ("", "", ""), status
      Note: The worker appends empty rows — so filter in worker OR:
      Better approach: yield the status only on the last real row.
      Revised: after the loop, use a return-style approach.

      REVISED IMPLEMENTATION:
      Build the full list first in the thread (acceptable since graph.build() is the
      slow part), then yield rows:

      rows = []
      for component in islands:
          group_size = len(component)
          detail = "Isolated" if group_size == 1 else f"Group of {group_size}"
          for handle in sorted(component):
              person = db_wrap.get_person_from_handle(handle)
              if person is None:
                  continue
              name = db_wrap.format_name(person)
              rows.append((name, str(group_size), detail))

      total_people = len(rows)
      total_groups = len(islands)
      status = f"{total_people} people in {total_groups} groups (max size: {self._max_size})"

      for i, row in enumerate(rows):
          # Yield status only on last row so worker shows it at completion
          s = status if i == len(rows) - 1 else ""
          yield row, s

      if not rows:
          yield ("", "", ""), f"No islands found (max size: {self._max_size})"

    Note on empty sentinel row: the worker appends ("", "", "") — this creates a
    blank row. To avoid this, modify the yield for the last row to use the real row
    with the status attached (as shown above).

    Avoid: Gtk calls inside _scan_fn — it runs in a background thread.
    Avoid: storing scan results on self from the thread — use the yield pattern only.
