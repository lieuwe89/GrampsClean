## GRAMPS Plugin Architecture Notes
- GRAMPS Tool plugins live in a subdirectory under the user's GRAMPS plugins folder
  (typically `~/.gramps/gramps<version>/plugins/grampsclean/`)
- The `.gpr.py` file is discovered by GRAMPS Plugin Manager on startup
- A Tool plugin class must extend both `gramps.gen.plug.Tool` and
  `gramps.gui.managedwindow.ManagedWindow`
- Tool.__init__ signature: `(self, dbstate, user, options_class, name, *args)`
- ManagedWindow requires a unique "track" list and a `build_menu_names()` method
- No OptionsTool needed at this stage — use a stub options class
- GTK3 is used directly (import Gi, require_version, then from gi.repository import Gtk)
- GRAMPS passes `user` (not `uistate`) as the second arg in newer GRAMPS 5.x Tool API;
  `uistate` is accessed as `user.uistate`
</context>

<task type="auto">
  <name>Task 2: Create tool skeleton with placeholder dialog</name>
  <files>grampsclean/tool.py</files>
  <action>
    Create grampsclean/tool.py with two classes:

    **GrampsCleanOptions** — stub options class:
    - Extends gramps.gen.plug.MenuToolOptions
    - __init__(self, name, dbstate=None): call super().__init__(name, dbstate)
    - add_menu_options(self, menu): pass  (no options yet)

    **GrampsCleanTool** — main tool class:
    - Extends both gramps.gen.plug.Tool and gramps.gui.managedwindow.ManagedWindow
    - __init__(self, dbstate, user, options_class, name, *args):
        * Call Tool.__init__(self, dbstate, options_class, name)
        * Call ManagedWindow.__init__(self, user.uistate, [], self.__class__)
        * Build and show a Gtk.Dialog:
          - Title: "GrampsClean — Data Quality Tools"
          - Set default size: 800 x 600
          - Add a Gtk.Label in the content area: "GrampsClean v0.1 — placeholder"
          - Add a Close button to the action area that calls self.close()
        * Call self.show() at the end
    - build_menu_names(self, obj): return ("GrampsClean", "GrampsClean")
    - close(self, *args): call ManagedWindow.close(self)

    Use gi.require_version("Gtk", "3.0") before importing Gtk.
    Avoid: Gtk.main() or any standalone event loop — GRAMPS manages the GTK main loop.
    Avoid: Dialog.run() — use show_all() and connect signals instead.
  </action>
  <verify>python3 -m py_compile grampsclean/tool.py</verify>
