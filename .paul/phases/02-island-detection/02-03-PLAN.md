<task type="auto">
  <name>Task 1: Extend ResultList with hidden tag column and on_activate callback</name>
  <files>grampsclean/widgets.py</files>
  <action>
    Add a hidden tag column (last column in ListStore, never rendered) and optional row-activation callback.

    1. Store `on_activate` callback and visible column list:
       ```python
       def __init__(self, columns, on_activate=None):
           self._visible_columns = columns
           self._on_activate = on_activate
       ```

    2. Create ListStore with N+1 columns (visible cols + 1 hidden str for tag):
       ```python
       self.store = Gtk.ListStore(*[str for _ in columns], str)
       ```
       Column renderers bind only to indices 0..N-1 as before — the hidden tag is index N.

    3. If `on_activate` is provided, connect the row-activated signal:
       ```python
       if on_activate:
           self.tree.connect("row-activated", self._row_activated)
       ```

    4. Add `_row_activated` handler:
       ```python
       def _row_activated(self, tree, path, column):
           model = tree.get_model()
           it = model.get_iter(path)
           tag = model.get_value(it, len(self._visible_columns))
           if tag and self._on_activate:
               self._on_activate(tag)
       ```

    5. Update `append_row` to pad to full ListStore width:
       ```python
       def append_row(self, *values):
           row = list(values)
           n_store_cols = self.store.get_n_columns()
           while len(row) < n_store_cols:
               row.append("")
           self.store.append(row[:n_store_cols])
       ```
       This keeps all existing callers (placeholder tabs, future tabs) working without changes — they pass N values, the hidden tag is auto-filled with "".

    Avoid: adding a `tag` keyword argument — `ScanWorker` calls `append_row` via `GLib.idle_add(result_list.append_row, *row_tuple)` which passes all args positionally. Including the handle as the last element of `row_tuple` is the correct approach.
  </action>

  <done>AC-2 satisfied: ResultList accepts on_activate callback; append_row pads automatically so existing callers are unaffected</done>
</task>
