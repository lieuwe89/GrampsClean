<objective>
## Goal
Build a kinship graph from the GRAMPS database — a data structure where every person is a node and every family relationship (parent-child, spouse) is an edge — that can be analyzed for connected components in Plan 02-02.

## Purpose
Island detection requires knowing which people are reachable from which other people via family links. A graph representation makes this trivial: connected components = groups, isolated nodes = single islands. This plan builds and validates that graph; the analysis runs in 02-02.

## Output
- `grampsclean/graph.py` — `KinshipGraph` class: builds an adjacency structure from the GRAMPS database using GrampsDb, exposes connected-component analysis
</objective>

<task type="auto">
  <name>Task 1: Implement KinshipGraph with stdlib union-find</name>
  <files>grampsclean/graph.py</files>
  <action>
    Create grampsclean/graph.py with one class: `KinshipGraph`.

    Use a **union-find (disjoint set)** data structure — no NetworkX needed.
    This resolves the NetworkX availability blocker from STATE.md.

    **KinshipGraph:**

    __init__(self, db_wrap):
    - Store `self.db_wrap = db_wrap`
    - `self._parent = {}` — union-find parent dict {handle: handle}
    - `self._rank = {}` — union-find rank dict {handle: int}
    - `self._built = False`

    **_find(self, handle) -> handle:**
    - Path-compressed find: if _parent[handle] != handle, recurse and compress
    - If handle not in _parent: initialise it first (_parent[h] = h, _rank[h] = 0)
    - Return root handle

    **_union(self, h1, h2):**
    - Find roots of h1 and h2
    - If same root: return (already connected)
    - Union by rank: attach smaller-rank tree under larger-rank root
    - If equal rank: increment winning root's rank

    **build(self):**
    - Iterate all people: for each person, call _find(person_handle) to register node
    - Iterate all families via db_wrap.iter_families():
      - Get father_handle = family.get_father_handle()
      - Get mother_handle = family.get_mother_handle()
      - Get child_handles = [ref.ref for ref in family.get_child_ref_list()]
      - Collect all non-None handles in this family into a list
      - Union them all pairwise with the first handle:
        for h in handles[1:]: self._union(handles[0], h)
    - Set self._built = True

    **get_components(self) -> list[set]:**
    - If not built: call build() first
    - Group all handles by their root (_find result)
    - Return list of sets (one set per component)
    - Every handle must appear in exactly one set

    **get_islands(self, max_size=10) -> list[set]:**
    - Get all components
    - Filter to those with len(component) <= max_size
    - Sort by len ascending (smallest isolated people first)
    - Return filtered sorted list

    No imports beyond Python stdlib. No GTK. No GRAMPS imports
    (receives db_wrap which wraps the GRAMPS db).

    Avoid: building a full adjacency list — union-find is O(α(n)) and
    uses far less memory than storing all edges explicitly.
  </action>
