    **1d. Add `_on_apply` method** — place it in the "Signal handlers" section, after `_on_cancel`:
    ```python
    def _on_apply(self, btn):
        """Write checked corrections to the GRAMPS database inside a single DbTxn."""
        findings = self._result_list.get_selected_findings()
        if not findings:
            self._result_list.set_status("No rows selected.")
            return

        db = self.db_wrap.db
        with DbTxn("Fix name prefixes", db) as trans:
            for f in findings:
                person = db.get_person_from_handle(f["handle"])
                if person is None:
                    continue
                name_obj = person.get_primary_name()
                surname_list = name_obj.get_surname_list()
                if not surname_list:
                    continue
                sn_obj = surname_list[0]

                if f["field_type"] == "given":
                    name_obj.set_first_name(f["proposed_value"])
                    sn_obj.set_prefix(f["prefix_found"])
                else:  # "surname"
                    sn_obj.set_surname(f["proposed_value"])
                    sn_obj.set_prefix(f["prefix_found"])

                db.commit_person(person, trans)

        count = len(findings)
        self._result_list.set_status(
            f"{count} correction(s) applied \u2014 use Edit \u203a Undo to reverse"
        )
    ```

    **Rules:**
    - `self.db_wrap.db` is the raw GRAMPS `DbWriteBase` passed in from `tool.Tool.self.db` — it exposes `get_person_from_handle()` and `commit_person()`
    - All mutations happen on the main GTK thread (button click handler) — no threading concern
    - One `DbTxn` wraps all commits — produces a single undo entry in GRAMPS history
    - `person is None` guard handles stale handles (person deleted between scan and apply)
    - `surname_list` empty guard is a safety check; practically all persons have at least one surname entry
    - Do NOT call `self._result_list.clear()` after apply — user may want to inspect what changed; a new scan is user-initiated
