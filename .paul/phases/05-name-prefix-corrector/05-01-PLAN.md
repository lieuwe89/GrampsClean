<acceptance_criteria>

## AC-1: Prefix detected in given name
```gherkin
Given a person whose given name begins with a known prefix (e.g. "van Jan")
  And the surname prefix field is empty
When a scan completes
Then the person appears as a finding with issue "Prefix 'van' in given name"
```

## AC-2: Prefix detected in surname
```gherkin
Given a person whose surname begins with a known prefix (e.g. surname="van Berg")
  And the surname prefix field is empty
When a scan completes
Then the person appears as a finding with issue "Prefix 'van' in surname"
```

## AC-3: Correctly placed prefix is not flagged
```gherkin
Given a person whose surname prefix field already contains the prefix (e.g. prefix="van")
When a scan completes
Then that person does NOT appear in the findings
```

## AC-4: Prefix list is configurable
```gherkin
Given the user opens the "Prefix Settings" expander
When the user edits the prefix list (one prefix per line)
  And clicks Scan
Then the scan uses the updated prefix list, not the default one
```

## AC-5: Navigation works on double-click
```gherkin
Given the result list contains a finding row
When the user double-clicks that row
Then GRAMPS navigates to that person in the People view
```

## AC-6: No-findings case handled gracefully
```gherkin
Given a scan completes with zero prefix issues found
When the result list is shown
Then the list is empty and the status bar reads "No prefix issues found."
```

  <name>Task 1: Create tab_prefixes.py with detection engine</name>
  <files>grampsclean/tab_prefixes.py</files>
  <action>
    Create `grampsclean/tab_prefixes.py` from scratch following the same structure as `tab_impossibilities.py`.

    **Module docstring:** Same threading strategy note — SQLite restriction, main-thread snapshot, pure-Python worker.

    **Imports:**
    ```python
    import gi
    gi.require_version("Gtk", "3.0")
    from gi.repository import Gtk, GLib

    from widgets import ResultList
    from worker import ScanWorker
    ```

    **DEFAULT_PREFIXES** — module-level list of str, sorted longest-first (important: longer prefixes must be checked before shorter ones to avoid "van" matching before "van der"):
    ```python
    DEFAULT_PREFIXES = [
        "van der", "van den", "van de", "van het",
        "in het", "op de", "in de", "aan de",
        "van", "de", "den", "het", "ten", "ter", "te",
        "von der", "von den", "von", "zum", "zur", "zu",
        "de la", "de los", "de las", "du", "des", "le", "la", "les",
    ]
    ```

    **`_build_snapshot(db_wrap)` function** (module-level, must run on main thread):
    - Iterates `db_wrap.iter_people()`
    - For each person, extract:
      - `handle = person.get_handle()`
      - `name_obj = person.get_primary_name()`
      - `given = (name_obj.get_first_name() or "").strip()`
      - surname_list = `name_obj.get_surname_list()`
      - If surname_list is non-empty: `sn_obj = surname_list[0]`; `surname = (sn_obj.get_surname() or "").strip()`; `prefix_field = (sn_obj.get_prefix() or "").strip()`
      - Else: `surname = ""`; `prefix_field = ""`
      - `display_name = db_wrap.format_name(person)` (use db_wrap helper — avoid duplicating formatting logic)
      - `gramps_id = db_wrap.get_gramps_id(person)`
    - Return dict: `{handle: {"name": display_name, "gramps_id": gramps_id, "given": given, "surname": surname, "prefix": prefix_field}}`
    - Avoid: accessing GRAMPS objects outside this function — all DB access must happen here on the main thread

    **`detect_issues(snapshot, prefixes)` function** (module-level, pure Python — safe for background thread):
    - `prefixes_lower` = `[p.lower() for p in prefixes]` — pre-normalise once
    - Iterate `snapshot.items()`:
      - Skip if `p["prefix"]` is non-empty (already correctly placed)
      - `given_lower = p["given"].lower()`
      - `surname_lower = p["surname"].lower()`
      - For each prefix in `prefixes_lower` (iterate in order — longest-first is already guaranteed by DEFAULT_PREFIXES):
        - If `given_lower.startswith(prefix + " ")`: issue = `f"Prefix '{prefix}' in given name"`; break
        - Elif `surname_lower.startswith(prefix + " ")`: issue = `f"Prefix '{prefix}' in surname"`; break
      - If an issue was found: append `{"handle": handle, "name": p["name"], "gramps_id": p["gramps_id"], "issue": issue}` to findings
    - Sort findings by `["name"]` for consistent display
    - Return findings list

    **`PrefixesTab(Gtk.Box)` class:**

    `__init__(self, db_wrap, uistate)`:
    - `Gtk.Box.__init__(self, orientation=Gtk.Orientation.VERTICAL, spacing=6)`
    - `self.set_border_width(8)`
    - Store db_wrap, _uistate, _snapshot=None, _worker=None
    - `self._build_toolbar()`
    - `self.pack_start(self._build_prefix_panel(), False, False, 0)`
    - `COLUMNS = [("Name", 0), ("ID", 1), ("Issue", 2)]`
    - `self._result_list = ResultList(COLUMNS, on_activate=self._navigate_person)`
    - `self.pack_start(self._result_list, True, True, 0)`

    `_build_toolbar(self)`: Scan + Cancel buttons — identical pattern to ImpossibilitiesTab._build_toolbar

    `_build_prefix_panel(self)`:
    - `expander = Gtk.Expander(label="Prefix Settings")`; `expander.set_expanded(False)`
    - Inside: a Gtk.Box (vertical, spacing=4) with:
      - `Gtk.Label(label="Prefixes to detect (one per line):")` — halign=START
      - `Gtk.ScrolledWindow` (min_content_height=80, policy=AUTOMATIC/AUTOMATIC, ShadowType.IN)
        - containing `self._prefix_view = Gtk.TextView()`
        - `self._prefix_view.set_wrap_mode(Gtk.WrapMode.WORD)`
      - Set initial text: `self._prefix_view.get_buffer().set_text("\n".join(DEFAULT_PREFIXES))`
    - Return expander

    `_get_prefixes(self)` (private helper):
    - Reads the TextView buffer text
    - Splits by newline, strips each, filters empty strings
    - Returns list of str (preserves user order — longest-first if using DEFAULT_PREFIXES)

    `_on_scan(self, btn)`:
    - Guard if worker running
    - `self._result_list.clear()`
    - `self._result_list.set_status("Reading database…")`
    - `self._snapshot = _build_snapshot(self.db_wrap)` — main thread
    - `self._active_prefixes = self._get_prefixes()` — capture now on main thread
    - Set scan_btn insensitive, cancel_btn sensitive
    - `self._worker = ScanWorker(self._result_list, self._scan_fn, None)`
    - `self._worker.start()`
    - `GLib.timeout_add(200, self._check_done)`

    `_on_cancel(self, btn)`, `_check_done(self)`: identical pattern to ImpossibilitiesTab

    `_navigate_person(self, handle)`: identical pattern to ImpossibilitiesTab._navigate_person

    `_scan_fn(self, _db_wrap_unused)`:
    - `findings = detect_issues(self._snapshot, self._active_prefixes)`
    - If not findings: `yield ("", "", ""), "No prefix issues found."`; return
    - `total = len(findings)`; `status = f"{total} prefix issue(s) found"`
    - For each finding: `row = (f["name"], f["gramps_id"], f["issue"], f["handle"])`
    - `yield row, (status if last else "")`

    Note: row is a 4-tuple — ResultList will unpack and pass (name, gramps_id, issue, handle) to append_row. The last element (handle) is the hidden tag used by _navigate_person on double-click.

    Avoid: calling GTK from the background thread; accessing GRAMPS DB in _scan_fn
