<objective>
## Goal
Add a threshold configuration panel to ImpossibilitiesTab so users can adjust the five age/timing thresholds before scanning, and wire the scan to use those values instead of DEFAULT_THRESHOLDS.

## Purpose
Configurable thresholds are a stated Phase 4 deliverable. Hard-coded limits force users to accept defaults that may not match their research context (e.g., historical populations where 12yr mothers were more common). The configuration panel makes the tool useful across different database contexts.

## Output
- `ImpossibilitiesTab` gains a collapsible GTK Expander above the result list containing five labeled spinbuttons
- `_on_scan` captures spinbutton values into a dict on the main thread and passes them to the worker
- DEFAULT_THRESHOLDS continues to serve as initial values for all spinbuttons
</objective>
